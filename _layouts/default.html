Latest news

JRetrofit 1.1 was released on 19.5.2010. Changes:

  • Now requires Java 5 to run.
  • All APIs use generics now.
  • Added a reflection wrapper api (class Reflect).

Download it here.

JRetrofit 1.0 final version was released on 5.5.2009. There were no changes from 1.0rc1.

Download it here.

JRetrofit 1.0rc1 was released on 10.6.2007. Changes:

  • Fixed a classloader bug when retrofitting an object with an interface which comes from a different classloader than the target class.
  • Retrofitted objects can now be serialized and deserialized.
  • More parameter checks, better error messages for incorrect usage.

This is a release candidate version. If no new bugs are found, this version will be released later as 1.0.

JRetrofit 0.6 was released on 6.1.2007. New features:

  • Private inner classes can now be used as stubs.
  • Protected and package private methods can be used to "implement" a public method (this breaks Java inheritance rules, but it is sometimes very useful when dealing with legacy stuff).

JRetrofit 0.5, the initial public version, was released on 19.11.2006, including the handling of partial and complete retrofittings, method lookup caching, tutorials and documentation.
